## Upgrading Sequa

Follow these steps to upgrade your Sequa instance safely. Always back up your configuration and database first.

<Steps>
  <Step title="Backup Data" icon="database">
    Export your knowledge graph and settings.

    ```bash
    sequa export --output backup-$(date +%Y%m%d).json
    ```
  </Step>
  <Step title="Update Package" icon="download">
    Install the latest version based on your platform.
  </Step>
  <Step title="Run Migrations" icon="settings">
    Apply database schema changes.

    ```bash
    sequa migrate up
    ```
  </Step>
  <Step title="Verify Installation" icon="check-circle">
    Test core functionality.
  </Step>
</Steps>

<Tabs>
  <Tab title="npm" icon="package">
    <CodeGroup tabs="Update,Verify">
      ```bash
      npm update @sequa/core@latest
      npm run build
      ```
      ```bash
      sequa --version
      curl http://localhost:3000/health
      ```
    </CodeGroup>
  </Tab>
  <Tab title="Docker" icon="docker">
    <CodeGroup tabs="Update,Verify">
      ```bash
      docker pull sequa:latest
      docker-compose up -d
      ```
      ```bash
      docker logs sequa-container
      curl http://localhost:3000/health
      ```
    </CodeGroup>
  </Tab>
</Tabs>

<Expandable title="Deprecations and Migration Guide" default-open="false">

Review deprecated features to ensure compatibility.

| Deprecated | Replacement | Migration Notes |
|------------|-------------|-----------------|
| `apiEndpoint` | `baseUrl` | Update in `sequa.config.json` |
| Legacy auth v1 | OAuth 2.0 | Run `sequa auth migrate` |

For v1.1.0 breaking changes:

````javascript
// Old config
{
  "apiEndpoint": "https://api.sequa.dev"
}

// New config
{
  "baseUrl": "https://api.sequa.dev"
}
````

</Expandable>
